# Keyboard Shortcuts

This page is the quick orientation to working faster with the keyboard. For the full shipped binding list, use [Keyboard Shortcuts Reference](../reference/keyboard-shortcuts-reference).

## What Shortcuts Are Good For

Keyboard shortcuts matter most when you are:

- switching tools without leaving the canvas
- opening sidebars and panels directly
- making repeated alignment, distribution, or formatting changes
- moving or resizing selections precisely

If you only learn a small set first, start with the tools you use every session.

## Good First Shortcuts

| Shortcut | Why it matters |
| --- | --- |
| `Command+1` through `Command+7` | Switch tools without breaking flow |
| `Alt+1` through `Alt+0` | Jump straight to the relevant sidebar panel |
| `Command+S` | Save quickly during iterative edits |
| `Command+K` | Open the command palette when you know the action name but not where it lives |
| `Command+=`, `Command+-`, `Command+0` | Control the viewport without reaching for menus |
| `Command+D` | Duplicate selected elements |

## Learn Shortcuts By Task

### Drawing And Switching Context

Tool shortcuts are useful when you are alternating between moving, drawing nodes, adding text, and connecting elements.

### Precise Cleanup

Arrow-key movement, grid-step movement, alignment, and distribution shortcuts are worth learning once your diagrams become dense enough that small adjustments take real time.

### Panel-Heavy Workflows

If you spend time in search, data binding, comments, or style editing, the sidebar shortcuts are often more valuable than shape-tool shortcuts.

## Discoverability In The App

- Tooltips and menus show resolved keybindings for many actions.
- The command palette is useful when you remember the action but not the shortcut.
- Some windows expose their shortcut directly through the sidebar action tooltip.

## Scope

- This page is intentionally selective.
- The complete lookup table lives in [Keyboard Shortcuts Reference](../reference/keyboard-shortcuts-reference).
