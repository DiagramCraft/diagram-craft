# Automatic Layouts

Diagram Craft provides five automatic layout commands in the selection context menu under **Layout**. Each command opens a dialog where you can preview and configure the result before applying it.

## The Available Layout Approaches

Diagram Craft currently exposes five graph layouts from the selection context menu:

| Layout | Best for | What to select |
| --- | --- | --- |
| Tree | clear hierarchies, ownership trees, dependency trees | one root node |
| Layered | directional flows, pipelines, step-by-step processes | the nodes you want arranged |
| Orthogonal | process and systems diagrams where right-angle routing reads better | the nodes you want arranged |
| Force-Directed | relationship maps and networks without one obvious top-to-bottom flow | the nodes in the connected area you want to explore |
| Series-Parallel | structured branching flows that are mostly sequential with controlled splits | the nodes you want arranged |

## Choose A Layout

- Start with **Tree** when the diagram has one obvious root and levels matter.
- Start with **Layered** when the diagram is mostly directional and should read left-to-right or top-to-bottom.
- Start with **Orthogonal** when you want a blockier process-map feel with right-angle connectors.
- Start with **Force-Directed** when you are still discovering the shape of a network.
- Start with **Series-Parallel** when the flow is structured but has branches that still need to stay tidy.

## Practical Constraints

The tools are not interchangeable:

- **Tree** expects a single selected root node.
- **Layered**, **Orthogonal**, and **Series-Parallel** work from the selected node set.
- **Force-Directed** starts from the selected nodes and expands across their connected component.

That means selection is part of the workflow. Before invoking layout, decide whether you want to reorganize the whole working area or just one local region.

## Apply Or Cancel The Preview

Changing a layout option updates the preview. Choose **Ok** to keep the new positions, or **Cancel** to restore the elements to their previous positions. Connector routes update with the arranged nodes.
