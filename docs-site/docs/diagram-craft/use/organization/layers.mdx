# Layers

Layers are the fastest way to control visibility and editing safety inside one diagram. Use them when the audience still needs one shared canvas, but you do not want every detail visible or editable at the same time.

## What Layers Are For

Treat layers as view management, not as document structure.

- Use a layer when content belongs to the same diagram but should be shown, hidden, or protected independently.
- Use a separate tab when the diagram needs a different scope or audience.
- Use a group when you want to move a local cluster together.

That distinction matters on larger diagrams. If a single canvas has baseline flow, annotations, review overlays, and optional detail, layers let you keep those concerns together without turning the canvas into noise.

## A Practical Layer Strategy

For a large working diagram, a reliable pattern is:

1. Keep the core flow on one regular layer.
2. Put optional callouts or supporting notes on separate layers.
3. Lock layers that should not move during cleanup passes.
4. Hide layers that are distracting while you work on one concern at a time.

This gives you one source diagram while still letting you narrow the editor to the part you are actively maintaining.

## When Layers Work Better Than Tabs

Choose a layer instead of a new tab when:

- the same elements must stay spatially aligned with each other
- reviewers need to compare multiple concerns in the same canvas
- the extra content is optional detail rather than a separate viewpoint

Choose a new tab instead when the diagram is becoming hard to read even after hiding non-essential layers.

## Lock And Hide Aggressively

Locking is useful during maintenance, not just at the end:

- lock stable background structure before a manual refinement pass
- hide annotation or review layers while running cleanup actions
- reorder layers when labels, highlights, or reference material need to sit visually above the main flow

If you are constantly selecting the wrong elements, that is usually a sign that the diagram needs more layer discipline.

## Rule And Reference Layers

The layer stack can hold more than plain drawing layers:

- **Reference layers** help reuse context from another diagram without copying everything into the current canvas.
- **Rule layers** are useful when visibility or styling depends on diagram data or query results.

Those are advanced tools, but they still fit the same idea: keep one diagram manageable by separating responsibilities.
