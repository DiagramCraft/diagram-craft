# Text and Labels

Text gives the diagram its meaning. Shapes and connectors establish structure, but the labels are what let another person understand the subject without guessing.

## Label Shapes As You Place Them

A good default habit is to name a shape immediately after adding it to the canvas. This keeps the diagram understandable even while the layout is still rough.

Short labels work best:

- prefer `API Service` over a full sentence
- use consistent naming for similar elements
- keep enough space inside the shape so the text is easy to scan

If a label starts to wrap awkwardly, resize the shape or shorten the wording before adding more detail elsewhere.

## Use Standalone Text For Context

Standalone text is useful when the information does not belong to a single shape. Common examples include:

- a short diagram title
- a note about assumptions
- a section heading for part of a larger canvas

Use standalone text sparingly. It should add context, not compete with the main flow.

## Edit Text In Place, Then Refine

A practical text workflow is:

1. Enter the shortest useful label first.
2. Adjust the element size or position so the label sits comfortably.
3. Refine text styling only after the wording and layout are stable.

This keeps the diagram readable early and avoids redoing formatting while labels are still changing.

## Keep Text Readable At A Glance

Before you move on:

- zoom out once to check whether the labels still read clearly
- make sure note text does not overpower the main diagram
- keep similar labels similar in tone and length

If the reader has to decode the wording, the diagram is carrying too much text.
