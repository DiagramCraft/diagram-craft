# Shapes and Elements

Shapes are the core building blocks of a diagram. In practice, you will usually add them from the left sidebar, where you can browse shape libraries, drag shapes onto the canvas, search enabled packages, and switch to icon browsing when needed.

## Use The Shape Tab

The **Shape** tab in the left sidebar is the main place to add diagram elements.

- open the **Shape** tab to browse the active shape libraries for the current document
- drag a shape from the picker onto the canvas to create a new node
- keep the picker open while building out a diagram so you can add several elements in sequence

This is the fastest way to build up the first structure of a diagram before you start refining labels, connectors, and styling.

## Use The Insert Shape Toolbar Action

The main toolbar also includes an **Insert Shape** action that opens a dialog with **Recent**, **Search**, and **Icons** tabs.

Use this when you want to insert an element from a centered picker instead of working from the left sidebar.

## Search And Browse The Picker

The picker supports a few different ways to find the element you need:

- browse visually through the active packages when you are exploring available shapes
- use the picker search when you know roughly what you want and do not want to scan the whole list
- switch between list and grid browsing from the picker settings menu, depending on whether names or thumbnails are more useful for the current package

The search works across the enabled packages, and the settings menu can expand that search behavior when you need a wider search surface.

## Choose Which Shape Libraries Are Active

The shape picker is only as useful as the package set behind it, so it is worth keeping the active libraries focused.

- use the picker settings menu to manage which packages are active for the current document
- keep only the libraries you need for the diagram you are working on, so the picker stays easier to scan
- add notation-specific packages when the diagram needs a formal shape language such as UML, BPMN, or C4

Active packages are document-scoped, so the package set travels with the document instead of being only a local preference.

For package guidance and the available built-in libraries, see [Built-in Stencils](../../stencils-templates/built-in-stencils).

## Use Recent, Model, And Icon Tabs

The same left sidebar window also exposes other tabs that are useful while adding elements:

- **Recent** lets you quickly reuse shapes you added earlier in the same editing session
- **Model** is for model-backed objects and provider-backed workflows; see [Data Sources](../data-integration/data-sources) in the **Data and Integration** section
- **Icon** lets you search and insert icons when a small symbol is more appropriate than a full diagram shape

If you enable optional icon libraries or vendor packages, the **Icon** tab is where those assets become most useful.
