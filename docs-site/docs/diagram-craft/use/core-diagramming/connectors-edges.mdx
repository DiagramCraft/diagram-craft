# Connectors and Edges

Connectors, also known as edges, are the lines and arrows between diagram elements. You can create them with the edge tool, by dragging from a shape anchor, or by using keyboard-driven linked-node actions when you want to build a connected structure quickly.

## Use The Edge Tool

The main toolbar includes the **Edge tool** for drawing connectors directly on the canvas.

- choose the **Edge tool** from the toolbar when you want to start a connector explicitly
- use the default shortcut `Command+3` when switching tools from the keyboard
- click or drag on the canvas to start the edge, then place the other end where you want it

An edge does not have to connect to two nodes. You can leave one or both ends free on the canvas when the line is acting as a loose annotation, callout, or partially connected draft.

## Drag From Shape Anchors

You can also start a connector directly from an existing shape.

- hover a shape and drag from one of its anchor points to begin a connection
- drop onto another shape to create a connected edge
- drop onto empty canvas space if you want a free end instead of a node-to-node connection

This is usually the fastest way to connect elements that are already on the canvas.

## Adjust Routing And Endpoints

After creating an edge, you can refine how it travels through the diagram.

- select the edge to expose its editable path and routing handles
- move endpoints when you want the edge to leave or enter a different side of a shape
- adjust the route when you need to avoid another element or create a cleaner path
- keep using free endpoints when the connector should point into open space rather than snap to a node

The current screenshot shows an edge selected with its routing handles visible.

## Use Keyboard Shortcuts For Connected Creation

Diagram Craft also exposes keyboard-driven actions that create connected structure from the current node selection.

- `Alt+Control+Arrow Up/Down/Left/Right` creates a linked node in that direction
- `Alt+Control+Shift+Arrow Up/Down/Left/Right` creates a linked node in that direction and keeps the current node selected
- these actions are useful when you are building out flows, trees, or neighboring steps without switching back to the shape picker after every placement

For the full shipped bindings, see [Keyboard Shortcuts Reference](../../reference/keyboard-shortcuts-reference).

## Direction And Arrow Style

When the connector represents flow, dependency, or direction:

- use arrowheads to make the direction explicit
- keep the route simple enough that the source and target are still easy to read
- change styling after the structural connection is correct, not before
