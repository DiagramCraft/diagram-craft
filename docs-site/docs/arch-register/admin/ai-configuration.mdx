## What You Configure

The current UI exposes configuration for:

- whether AI is enabled
- provider selection
- API key or equivalent credentials
- model
- temperature
- optional system prompt

## Operational Notes

The workspace prompt is shared configuration, not a per-user preference. Treat it as a workspace behavior control and
manage it accordingly.

Workspace API keys are stored using a versioned encrypted format. Configure `AI_ENCRYPTION_KEY` before saving a key;
new plaintext credential writes are rejected. For legacy values or key rotation, run the server's
`rotate:ai-keys` command in `--check` mode and then `--apply` mode with the appropriate `--legacy-format` and old-key
environment variables. Do this during a maintenance window, and remove the old key after successful migration.
