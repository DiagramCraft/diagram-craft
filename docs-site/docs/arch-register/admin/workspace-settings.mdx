## Covered Areas

The implemented settings area includes:

- general workspace identity
- lifecycle
- model overview
- schemas
- document types and templates
- roles and permissions
- teams
- members
- AI
- export and import
- analytics
- audit log
- danger zone

## General

Use general settings to manage the basic workspace identity such as name, slug, description, and visual identity.

## Lifecycle

Lifecycle settings define the allowed lifecycle states for entities in the workspace. They control classification and
filtering, not access.

## Document Types And Templates

Document Types & Templates defines the structured Markdown documents available in the workspace. Document types
provide fields and validation rules; templates provide reusable Markdown bodies and metadata defaults. Project owners
can also create project-specific Markdown templates from project settings.

See [Document Types And Templates](document-types-templates) for the full configuration and usage workflow.

## Analytics, Audit, And Danger Zone

These three sections exist for different operational jobs:

- **Analytics** for workspace-wide coverage and completeness views
- **Audit log** for change inspection
- **Danger zone** for irreversible operations

Keep them conceptually separate. They solve different problems.
