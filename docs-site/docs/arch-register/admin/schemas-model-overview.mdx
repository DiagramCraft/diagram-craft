# Schemas And Model Overview

This area defines the structure of the architecture catalog.

## Schemas

Schemas define:

- entity types
- fields
- enums
- references
- containment relationships

Supported field types include text, long text, boolean, date, select, reference, and containment.

## Modeling Guidance

Use:

- reference fields for cross-cutting links
- containment fields for parent-child hierarchy

That distinction matters because containment drives hierarchical views while references drive relationship views.

## Model Overview

Model Overview is the visual companion to schema editing. Use it to inspect how entity types connect across the
workspace rather than only reading form-based definitions.
