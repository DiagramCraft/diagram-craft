# Workspaces

Workspaces are the top-level boundary for architecture data.

Use a workspace to separate schemas, entities, projects, teams, and settings for one product or domain.

## Common actions

- Switch between workspaces from the top bar.
- Create a new workspace from the workspace selector.
- Update workspace details in Settings.
- Use workspace roles and teams to control access.
- Add shared documentation with [workspace wiki pages](wiki).
