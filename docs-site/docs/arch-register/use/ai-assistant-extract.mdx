# AI Assistant And Extract

Arch Register exposes two AI-facing sections in the workspace shell: **Assistant** and **Extract**. Document types
can additionally define AI actions that run directly from a document.

## Assistant

Assistant is the conversational workspace screen. Use it when you need:

- help reasoning about workspace data
- guided exploration across entities and projects
- AI-supported drafting with workspace context

Assistant behavior depends on workspace AI configuration, including provider, model, and optional system prompt.

## Extract

Extract is the structured AI flow for turning input material into more usable architecture data. Treat it as a focused
workflow rather than a general chat interface.

## Document AI Actions

Document types can define interactive AI actions. When a document's type has actions configured, anyone who can view
the document can launch them from the document sidebar.

An action runs its predefined prompt read-only against the document's body, structured metadata, type, and location —
it never modifies the document. The result is temporary; if it is worth keeping or exploring further, continue it as
an Assistant conversation from the result panel.

Actions are defined per document type in
[Document Types and Templates](../admin/document-types-templates).

## Availability

These sections are workspace features, not guaranteed defaults. If they are missing, check:

- whether AI is enabled for the workspace
- whether your role exposes the feature
- whether the workspace has valid provider configuration
