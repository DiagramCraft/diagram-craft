# Content

Arch Register has a shared content model across workspace, project, and entity scope. This is where wiki pages,
diagrams, uploads, and folders live.

## Workspace Content

Use workspace content for shared diagrams and shared documentation.

## Project And Entity Content

Project content stays with one initiative. Entity content stays with one record. The content capabilities are similar,
but the parent boundary and permission context are different.

## Diagrams

Diagrams are part of the content model. They can live at workspace, project, or entity scope.

## Typed Markdown Documents

Markdown documents can optionally use a [document type](../admin/document-types-templates). A document type adds
structured properties such as workflow status (using an enum field), dates, numbers, and links to entities or other
documents. Required properties are validated when the document is saved, while expected properties produce a warning.

When creating a wiki page, select a document type and optionally a template. Templates can pre-fill the Markdown body
and structured metadata, including the `{{title}}` placeholder for the new document name. Workspace templates are
available across the workspace; project templates are available within their project.

Untyped Markdown remains available for documents that do not need structured properties. Existing typed documents
keep their metadata and type history when a document type is archived.

## Wiki Pages And Files

Use wiki pages for lightweight documentation close to the thing it describes. Use uploaded files when the source
material should stay intact rather than be rewritten into markdown.

## Choosing The Right Scope

Use:

- workspace content for shared standards and shared diagrams
- project content for initiative-specific documentation and files
- entity content for notes and files attached to one record
