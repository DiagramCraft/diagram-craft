# Entity Views

The entity browser supports several screens for different kinds of analysis. Start in the default table screen, then
switch when the question changes.

## Table

Use the table screen for auditing, sorting, and comparing fields side by side.

When you join an assessment, assessment fields can appear beside normal entity fields. This is useful for comparing
review scores, filtering by unanswered assessment fields, or building portfolio views from assessment answers.

## Bubble

Use bubble when you want to plot entities against two axes at once, with size and colour adding a third and fourth
dimension. X axis, Y axis, size, and colour can each be mapped independently to a numeric or categorical field —
numeric sources include schema number fields and a joined assessment's rating fields; categorical sources include
select fields, lifecycle, owner, and a joined assessment's enum fields. Entities that land on the same coordinate are
spread apart automatically so nothing is hidden behind another bubble.

## Map

Use map when you want a three-level containment map of your entities, with each box colored by a single configurable
metric. Pick a schema for each of up to three levels along containment relationships, then choose a metric — a
numeric field, lifecycle state, or a joined assessment's rating/enum fields — and an aggregation (count, sum,
average, minimum, maximum, or worst) to roll up over each box's descendants. Numeric metrics render as a colour
gradient normalized across the visible boxes; lifecycle metrics use the workspace's lifecycle colours; enum-based
metrics use the dominant option among descendants, with the full distribution available in the tooltip. Click a box
to focus the map on that entity and drill into its descendants; use the breadcrumb to navigate back up.

## Saved Views

View configuration — fields, filters, sorting, display mode, and joined data such as assessment fields — can be saved
and reused. Use saved views when the same slice of the catalog recurs in reviews, governance work, or planning
sessions.

Technology lifecycle work can use the same saved-view surfaces. Technology Release records can be displayed in a
table for support-date review, in a radar using technology category and Adopt/Trial/Assess/Hold status, or in a
timeline mapping release date to EOL date. These views show release-level information; filtering applications by the
support state of referenced releases requires relationship-aware reporting beyond the current entity filters. The
Technology Release catalog is included in the Default workspace template and seeded workspace; named model templates
retain their native technology concepts.

## Import And Export

The entity browser also supports CSV workflows:

- export the current browser slice to CSV
- import entities through the import screen, with review before anything is written
- download schema-specific import templates to prepare data outside Arch Register
