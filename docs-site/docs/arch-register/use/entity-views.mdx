# Entity Views

The entity browser supports several screens for different kinds of analysis. Start in the default table screen, then
switch when the question changes.

## Table

Use the table screen for auditing, sorting, and comparing fields side by side.

<ThemedImage
  alt="Entity table screen in Arch Register"
  sources={{
    light: require('/img/arch-register/entities/browser-overview-light.png').default,
    dark: require('/img/arch-register/entities/browser-overview-dark.png').default,
  }}
/>

When you join an assessment, assessment fields can appear beside normal entity fields. This is useful for comparing
review scores, filtering by unanswered assessment fields, or building portfolio views from assessment answers.

Use matrix when you need to inspect density, gaps, or overlap across a relationship dimension.

## Bubble

Use bubble when you want to plot entities against two axes at once, with size and colour adding a third and fourth
dimension. X axis, Y axis, size, and colour can each be mapped independently to a numeric or categorical field —
numeric sources include schema number fields and a joined assessment's rating fields; categorical sources include
select fields, lifecycle, owner, and a joined assessment's enum fields. Entities that land on the same coordinate are
spread apart automatically so nothing is hidden behind another bubble.
