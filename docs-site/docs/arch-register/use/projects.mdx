# Projects

Projects are the bounded work areas in Arch Register. They group metadata, files, related entities, and planned
change around one initiative.

## Milestones

Milestones give a project its planning checkpoints. Each milestone has a target date and a status, and planned entity
changes can be associated with a milestone instead of a fixed date.

## Planned Changes

Planned changes record proposed future changes to entities within the scope of a project, without touching the
current live entity state. A planned change can be:

- targeted at a future date, or associated with a project milestone
- inspected in the project timeline and in entity timeline views, alongside historical change
- applied when it is ready, promoting the proposed state into the live entity

Use planned changes when a project needs to describe a future state — a decommissioning, a migration target — before
it happens.

## Use Projects When

Use a project when the main thing you are organizing is a file and planning boundary rather than a catalog record.
Use assessments inside a project when the project needs structured review data across a scoped set of entities.
