# Entities

Entities are the catalog records in Arch Register. This page covers the core workflows: browse, create, edit, and
understand relationships.

## Browse Entities

The entity browser is the main catalog screen. Use it to:

- scan records by schema and lifecycle
- filter by owner or field values
- switch between saved working slices
- open a record into its detail view

## Create An Entity

The normal flow is:

1. open **Entities**
2. select **New entity**
3. choose the schema
4. provide a name and required fields
5. save the record

The created record opens in the detail screen.

Workspaces can also configure entity templates. Creating from a template pre-fills the record so recurring entity
structures stay consistent.

## Edit An Entity

The detail screen is where you edit:

- built-in metadata
- schema-driven fields
- owner and lifecycle
- target lifecycle and target date
- links and tags

The entity detail screen also exposes related content and history rather than forcing those concerns back into the
browser.

If the entity's schema requires approval for changes, saving submits a change proposal for review instead of updating
the record directly. See [Governance and Approvals](governance).

## Bulk Edit

In the table view of the entity browser, select multiple rows to open the bulk edit toolbar. Bulk edit updates
supported fields across the whole selection in one operation — useful for reassigning owners or correcting a
classification field across many records.

## Relationships

Entities can relate to each other in two main ways:

- **reference** fields for directional links such as depends on or provides
- **containment** fields for hierarchy such as parent-child structures

Use containment when one parent should own the hierarchy. Use reference when the relationship cuts across ownership or
structure.

## Content And History

Each entity can also carry content in its own scope:

- wiki pages
- diagrams
- uploaded files

Entity detail also exposes several context and planning tabs:

- **Topology** for the entity's relationships and dependencies in a topology layout
- **Graph** for exploring the relationship graph outward from the entity
- **Dependents** and **Related content** for records and material connected to the entity
- **Changes** for entity change history
- **Timeline** for historical and planned change over time
- **Discussions** for threads attached to the record

Use the watch button on the detail screen to follow the entity and receive
[notifications](collaboration-notifications) when it changes.

When an entity is in scope for an open project assessment, its detail screen also shows an **Assessments** tab. Use
that tab to fill in the entity's assessment fields without opening the full project assessment grid.
