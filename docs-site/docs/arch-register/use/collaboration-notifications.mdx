# Collaboration And Notifications

Arch Register keeps collaboration close to the records: watch an entity to follow it, discuss records and documents
where they live, and pick up everything in one notification feed.

## Watching Entities

Use the watch button on an entity detail screen to follow that record. Watched entities generate notifications when
relevant changes occur.

The bell menu in the top bar has a **Watching** tab that lists everything you watch, with a shortcut to unwatch
records you no longer care about.

## The Notification Feed

The bell in the top bar is the single consolidated in-app feed. It combines:

- changes on watched entities
- governance action items assigned to you

An unread count badge appears on the bell. From the feed you can:

- open a notification — entity notifications open the entity, governance notifications open [My Work](governance)
- clear an individual notification
- mark all notifications read

Governance action-item notifications clear themselves once the underlying task is resolved, superseded, or
cancelled — you do not need to open each one to keep the feed accurate.

## Notification Preferences

Control which notifications you receive from **Account settings → Notifications**. Preferences work per notification
type and per delivery channel:

- in-app delivery is on by default for normal notification types and off by default for reminder types
- email is deliverable when configured by the deployment; Slack and SMS remain unavailable

Preferences are scoped per user per workspace, and only affect future notifications — existing feed items are not
removed retroactively.

## Discussions

Discussion threads attach to supported records and content:

- entity detail screens include a **Discussions** tab
- supported wiki and document content supports inline comments and discussion threads
- the top bar exposes a discussions menu for threads you participate in

Use discussions when the conversation should stay attached to the object it is about, rather than living in an
external channel.
