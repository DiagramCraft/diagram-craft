# Account Settings

Account settings are personal, not workspace-governance settings.

## Current Scope

The implemented account area currently focuses on:

- profile details
- appearance
- notification preferences
- API tokens for service integrations

## Profile

Profile covers the identity data tied to the current sign-in, such as display name and sign-in provider details.

## Appearance

Appearance controls how your account is represented visually in the workspace, including avatar color choices.

## Notifications

The **Notifications** section controls which notification types you receive, per delivery channel. In-app delivery is
on by default for normal notification types and off by default for reminders. Email delivery is available when the
deployment has configured its email provider; Slack and SMS remain unavailable. Preferences apply per workspace and
only affect future notifications.

See [Collaboration and Notifications](collaboration-notifications) for how the notification feed works.

## API Tokens

API tokens are workspace-scoped credentials for tools and service integrations. Create them from the **API tokens**
section, choose a workspace and capability set, and copy the secret when it is shown. The complete token is shown
only once.

For an external AI client, grant `ws.view` for read access. Add `ent.edit` only when the integration must create or
update entities. Treat tokens like passwords and revoke them when they are no longer needed.

See [MCP Server](../reference/mcp-server) for connecting an AI client to the Arch Register catalog.

## Why This Is Separate

Account settings should not be confused with workspace settings:

- account settings belong to the signed-in user
- workspace settings belong to the workspace and affect other users
