# Advanced Entity Views

The entity browser supports more than one presentation. These views are for analysis, not just browsing.

## Browser Modes

The implemented modes are:

- **Table** for audits and side-by-side comparison
- **Cards** for quick scanning
- **Tree** for containment structures
- **Radar** for quadrant-based mapping
- **Timeline** for date-driven views and planned change context
- **Matrix** for relationship density and coverage
- **Bubble** for plotting entities across X/Y/size/colour dimensions at once
- **Explore** for configurable side-by-side exploration

## Saved Views

Saved views let teams preserve working slices of the catalog. Use them when the same filters or visual configuration
recur in reviews, governance work, or planning sessions.

## Import And Export

The entity area also includes:

- CSV export of the current browser slice
- CSV import through the import screen
- downloadable import templates tied to a schema

These are advanced operational workflows and should stay separate from the basic create/edit path.

## Timeline And Change Context

Entity history appears in more than one place:

- change history on the entity detail screen
- timeline views in the browser
- project-planned changes that feed date-oriented views

That split is intentional. Audit-style history answers what changed; timeline views answer how a record evolves over
time.
