# Projects And Diagrams

Projects are the project-scoped content containers in Arch Register. They group diagrams, wiki pages, files, folders,
and templates around a bounded area of work.

## What A Project Holds

Projects can contain:

- diagrams
- wiki pages
- uploaded files
- folders
- templates
- collaborative editing sessions

## Project Browser

Use the project browser to:

- scan draft, active, and archived projects
- pin important projects
- open the project detail view
- move into project-specific files and folders

Archived projects are separated from active work, which keeps the main navigation focused on current material.

## Project Detail

Project detail is where you manage:

- status and metadata
- owner
- pinned state
- file tree
- associated entities
- planned changes

This is the center of project-specific navigation, not just a metadata screen.

## Diagrams

New diagrams can start from:

- a blank canvas
- a project template
- a workspace template

Arch Register manages the file, location, and collaboration context. Diagram editing itself is handled by Diagram
Craft.

## Collaboration

When multiple users open the same diagram, Arch Register connects them to the same collaboration session. Presence and
live updates belong to the project file context, not just the editor.
