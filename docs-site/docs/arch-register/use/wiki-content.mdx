# Wiki And Content

Arch Register has a shared content model across workspace, project, and entity scopes. This page explains when to use
each scope and what capabilities are shared.

## Content Scopes

The three scopes are:

- **Workspace content** for shared diagrams and documentation
- **Project content** for project-specific files and notes
- **Entity content** for files and notes attached to one entity

Each scope supports folders, markdown pages, diagrams, and uploaded files. The important difference is parent context
and access behavior.

## Wiki Pages

Wiki pages are the lightweight documentation layer. Use them for:

- operational notes
- architecture rationale
- runbooks
- release notes
- design fragments that should stay close to the related object

## Attachments And Embedded Content

The markdown surface also supports:

- attachments
- embedded diagram references
- embedded entity references and MDX blocks
- revision history and compare flows

### Entity Graphs

Embed a compact, read-only dependency or impact graph for one entity with `EntityGraph`:

```mdx
<EntityGraph id="APP-001" depth="2" direction="both" />
```

`id` accepts an entity public ID or UUID. `depth` defaults to `1` and is limited to `1–3`; `direction` can be
`upstream`, `downstream`, or `both` (the default). Nodes are clickable, and a graph truncated by the selected depth
links to the entity’s full Graph tab.

This makes the wiki/content surface more than plain markdown editing.

## Choosing The Right Scope

Use:

- workspace content for shared standards and shared diagrams
- project content for initiative-specific documentation
- entity content for notes and files attached to a single record
