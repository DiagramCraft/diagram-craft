# Permission Model

This page summarizes the implemented permission model without repeating the full admin guides.

## Layers

1. Global permissions
2. Workspace roles
3. Team roles

## Practical Meaning

- global permissions control platform-wide administration
- workspace roles control workspace shell and governance capabilities
- team roles control owned entities and projects

## Important Separations

- lifecycle is classification, not permission
- ownership is not the same as workspace membership
- project access depends on the wider permission model; it is not an isolated project ACL system
