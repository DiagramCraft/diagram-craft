# Scope Matrix

This page is the quick reference for where content lives and which parent context applies.

| Scope | Best for | Parent boundary | Typical content |
| --- | --- | --- | --- |
| Workspace content | Shared docs and shared diagrams | Workspace | Standards, shared notes, common diagrams |
| Project content | Initiative-specific material | Project | Project wiki pages, project diagrams, uploads |
| Entity content | Record-specific context | Entity | Attached docs, diagrams, files, runbooks |

## Rule Of Thumb

- if the material belongs to the whole workspace, use workspace content
- if it belongs to one initiative, use project content
- if it belongs to one catalog record, use entity content
